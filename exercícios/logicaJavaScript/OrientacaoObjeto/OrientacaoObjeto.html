Progamação orientada a objetos

Programação Orientada a Objeto (também conhecida pela sua sigla POO) ou Modelagem Orientada ao Objeto, é um modelo/paradigma de projeto e programação de software baseado na abstração digital do mundo real, através da composição e interação entre diversas unidades chamadas de 'objetos' e as classes (representando objetos reais contendo identidade, propriedades e, métodos);[2] baseado em quatro principais componentes da programação: abstração digital, encapsulamento, herança e, polimorfismo

Dentre as várias abordagens da POO, as baseadas em classes são as mais comuns: usando um objeto que pode ser manipulado, criado a partir de uma classe através do instanciamento.[2] Estes possuem métodos que modificam seus próprios dados, definindo o tipo do objeto. A classe determina o comportamento (métodos), estados possíveis (atributos) e, o relacionamento com os outros objetos.[3] A maior alternativa as classes é o uso de protótipos, onde objetos são cópias de outros, onde apenas a herança simples é implementada pela cópia.

Pontos importantes

O que é POO

  • Programação orientada a objetos(POO): método de programação(paradigma) que usa tipos de dados personalizados, ou seja, podemos construir outros tipos de dados além dos dados primitivos.
  • Baseia-se fundamentalmente no conceito de Objetos

Vantagens de POO

  • Fornece uma estrutura modular para construções de programas. Os programas ficam divididos em partes menores o que facilita a organização.
  • O software se torna mais fácil de ler e posteriormente de fazer manutenção.
  • POO permite o reúso de códigos já programados (através das classes e as instâncias delas), o que torna o desenvolvimento mais rápido e produtivo
  • Os objetos instânciados podem ser reultilizados em aplicações diferentes
  • Encapsulamento: não é preciso conhecer a implementação interna de um objeto para poder usá-lo. Ou seja, podemos usar classes e objetos, trechos de códigos que outras pessoas desenvolveram sem saber com foram progamados internamente.

Abstração

Em ciência da computação, a abstração é a habilidade de concentrar nos aspectos essenciais de um contexto, ignorando características menos importantes ou acidentais. Em modelagem ou programação orientada a objetos, uma classe é uma abstração de entidades existentes no contexto de um software, como exemplo, imaginamos a abstração referente a classe Animais, que se divide em várias sub-classes, como: Anfíbios, Répteis e Mamíferos, que contém objetos de sub-classe como Ser-humano, Jacaré e outros.

Uma classe abstrata é desenvolvida para representar entidades e conceitos abstratos. A classe abstrata é sempre uma superclasse que não possui instâncias. Ela define um modelo (template) para uma funcionalidade e fornece uma implementação incompleta - a parte genérica dessa funcionalidade - que é compartilhada por um grupo de classes derivadas. Cada uma das classes derivadas, completa a funcionalidade da classe abstrata adicionando um comportamento específico.

Pontos importantes

  • Abstrair é selecionar aspectos específicos de um problema a ser analizado, deixando de lado outros aspectos. Representar uma entidade(algo que ocorre, que existe) do mundo real na forma de idéias.
  • Entidades abstraidas podem se comunicar entre si através de mensagens

Mensagens

Mensagem, em programação orientada a Objeto, seria uma espécie de 'mensagem fictícia' que ocorre dentro do sistema para 'invocar' diretamente uma classe com finalidade de usá-la ou simplesmente invocar 'estados' da classe. Resumindo, mensagem seria uma ativação de classe e/ou derivados.

Pontos importantes

  • Os objetos se comunicam através de troca de mensagens
  • Mensagem é um sinal enviado de um objeto a outro, requisitando um serviço, usando uma operação programada no objeto chamado.
  • As mensagens somente ocorre entre objetos que possuem uma associação entre si
  • As mensagens também são programasas
  • Quando uma mensagem é recebida, uma operação é invocada no objeto chamado.
  • Existem vários formatos de chamadas: procedures(subs e functions), passagem de sinais entre threads, acionamento de eventos, e etc.

Classes

Na programação orientação a objetos, uma classe é um Tipo abstrato de Dados (TAD). Ou seja, uma descrição que abstrai um conjunto de objetos com características similares (um projeto do objeto), é um código da linguagem de programação orientada a objetos que define e implementa um novo tipo de objeto, que terão características (atributos) que guardaram valores e, também funções específicas para manipular estes. Formalmente, é um conceito que encapsula abstrações de dados e procedimentos que descrevem o conteúdo e o comportamento de entidades do mundo real, representadas por objetos. De outra forma, uma classe pode ser definida como uma descrição das propriedades ou estados possíveis de um conjunto de objetos, bem como os comportamentos ou ações aplicáveis a estes mesmos objetos.

A classe é um elemento primordial de um diagrama de classes, modelagem importante na programação orientada a objetos.

Pontos importantes

  • Uma classe representa uma idéia ou conceito e classifica objetos que tenham propriedades similares.
  • São os blocos de construção mais importane do sistema Oritenção a Objetos, com responsabilidades bem definidas dentro da aplicação.
  • Coleção de objetos descritos com os mesmos atributos e operações.
  • Tipo personalizado de dados, è como um "molde" para criação de objetos.

Estrutura de Classe

Uma classe comumente define o estado e o comportamento de um objeto implementando atributos e métodos. Os atributos (por vezes referidos como "campos", "membros de dados" ou "propriedades"), indicam as possíveis informações armazenadas por um objeto de uma classe, representando o estado de cada objeto. Os métodos (por vezes referidos como "operações" ou serviços) são procedimentos que formam os comportamentos e ações oferecidos por objetos de uma classe, sendo responsáveis por alterar o estado ou fornecer informações sobre um objeto.

Outros elementos associados a uma classe são:

  • Construtor e destrutor - métodos especiais que definem o comportamento do objeto de uma classe no momento da sua criação e destruição
  • Propriedade - define o acesso a um estado do objeto.
  • Evento - define um ponto em que o objeto pode chamar outros procedimentos de acordo com seu comportamento e estado interno.
  • Um conjunto de classes são organizadas em um projeto só recebe o nome de Namespace.

Atributos

Os Atributos em Programação Orientada a Objetos são os elementos que definem a estrutura de uma classe. Os atributos também são conhecidos como variáveis de classe, e podem ser divididos em dois tipos básicos: atributos de instância e de classe.

Os valores dos atributos de instância determinam o estado de cada classe. Um atributo de classe possui um estado que é compartilhado por todos os objetos de uma classe. Atributos de classe podem ser chamados também de atributos estáticos ou dinâmicos. "Const" são considerados atributos de classe quando estão fora de qualquer método.

  • Atributos Estáticos: mantém o mesmo valor durante sua exitência, como uma data de nascimento por exemplo.
  • Atributos Dinâmicos: Valores que variam com o passar do tempo como a idade de uma pessoa.

As mensagens enviadas a um objeto (chamada de um método) podem mudar o valor de um ou mais atributos, alterando o estado de um objeto instânciado. Um atributo é um dado para o qual cada objecto instânciado tem seu próprio valor. Atributos são, basicamente, a estrutura de dados que vai representar a classe.

Métodos

Em orientação a objetos, uma Função (ou função membro) é uma sub-rotina (ou procedimento ou função) associada a um objeto, e que possui acesso aos seus dados, as variáveis membro. Ele é executado por um objeto ao receber uma mensagem. Os métodos determinam o comportamento dos objetos de uma classe e são análogos às funções ou procedimentos da programação estruturada. O envio de mensagens (chamada de métodos) pode alterar o estado de um objeto.

Em linguagens baseadas em classes eles são definidos na classe. Métodos definem o comportamento a ser exibido pelas instâncias da classe associada no tempo de execução do programa. Métodos possuem a propriedade especial que em tempo de execução, possuem acesso aos dados armazenados em uma instância da classe (ou instância de classe ou objeto de classe ou objeto) que estão associados e são, desta forma, capazes de controlar o estado da instância. A associação entre classe e método é chamada de ligação (binding). Um método associado com uma classe é dito estar ligado (bound) à classe. Métodos podem ser ligados a uma classe em tempo de compilação (ligação estática) ou a um objeto em tempo de execução (ligação dinâmica).

  • Método é a lógica contida em uma classe para atribuir comportamentos (sequência de comandos), e é indentificado por um nome que deve ser criado.
  • São similares as funções e procedimentos, ou seja, além de ter uma nome tem um corpo onde as intruções de comandos são declarados para serem executado pelo objeto quando for invocado (chamado).
  • O ato de invocar (chamar) um método é o que chamamos de passagem de mensagens para o objeto instânciado.
  • Geralmente os métodos são associados as ações que o objeto pode executar.
  • Os métodos podem receber dados para processamentos através dos parâmetros e depois retornar dados processados para a classe.
  • Um Método deve executar uma única operação de durante sua existência. Se for necessário realizar múltiplas operações em uma classe, devemos criar multiplos métodos para elas.

Objetos (Instância de uma classe)

Objeto é uma referência a um local da memória que possui um valor. Um objeto pode ser uma variável, função, ou estrutura de dados. Em programação orientada a objetos, a palavra objeto refere-se a um "molde"(classe) que passa a existir a partir de uma instância da classe, ou seja, é uma ocorrência de uma classe. O Objeto é uma entidade que formaliza o modo pelo qual comprendemos algo no domínio de um problema. É a entidade mais próxima possível das entidades do mundo real. Objetos são os substantivos do domínio de uma problema. A classe define o comportamento do objeto, usando atributos (propriedades) e métodos (ações)

Quais as características que existem em um Objeto em POO?

Objeto em Programção Orientada a Objetos tem Estado, Comportamento e uma Indentidade

  • Estado: é definido pelas propriedades que o objeto possui e os valores que são atribuidos a ele.
  • Comportamento: definido pela forma que o objeto reage, em termos de mudanças do seu estado e o relacionamento com outros objetos.
  • Indentidade: porpriedade pela qual ele se distingue dos outros.

O que diferrencia um objeto de uma classe?

  • Classe: a classe é como um objeto de "molde" de características que são passados para o objeto instânciado (criado), através de atributos com valores e metodos de comportamentos ou funções.
  • Objeto: já o objeto é uma entidade concreta com tempo e espaço de existência, enquanto que a classe é apenas uma abstração do que seria o objeto.

Qual a relação entre classe e objetos

Objetos também são entidades abstratas porém eles tem um significado concreto com relação ao problema que estivermos analizando. A classe irá originar os objetos somente por requsição, esse processo é chamado de instânciação do objeto, por isso em POO, os objetos são chamados de instância de classe. Na Programação orientada a objetos criamos as classes que por sua vez originam os objetos que serão chamados em tempo de execução. Sempre quando a aplicação for executa quem será chamado para a execução será o objeto instânciado, e não a classe. Uma classe pode originar múltiplos objetos apartir dela, porém com valores de atributos distintos entre eles.

Herança

Herança é um princípio de orientação a objetos, que permite que classes compartilhem atributos e métodos, através de "heranças". Ela é usada na intenção de reaproveitar código ou comportamento generalizado ou especializar operações ou atributos. O conceito de herança de várias classes é conhecido como herança múltipla.

Como exemplo pode-se observar as classes 'aluno' e 'professor', onde ambas possuem atributos como nome, endereço e telefone. Nesse caso pode-se criar uma nova classe chamada por exemplo, 'pessoa', que contenha as semelhanças entre as duas classes, fazendo com que aluno e professor herdem as características de pessoa, desta maneira pode- se dizer que aluno e professor são subclasses de pessoa. Também podemos dizer que uma classe pode ser abstrata(abstract) ou seja ela não pode ter uma instância, ela apenas "empresta" seus atributos e metódos como molde para novas classes.

Pontos importantes

  • Herança, em teoria, é um relacionamento entre as classes no qual uma subclasse "herda" os membros (atributos e métodos) de outra classe.
  • Com herança entre as classes, podemos criar classes mais complexas sem repetição de códigos.

Herança de classes com javaScript

Para que uma subclasse herde propriedades de uma classe pai quando programamos com javaScript devemos declarar ela com a seguinte sintaxe:

        
            class Carnivoro extends Mamiferos {
                
            }

Desta forma usando a palavra reservada "extends", no exemplo de sintaxe, a subclasse "Carnivoro" receberá de maneira direta as propriedades de dentro do "constructor" da classe pai "Mamiferos" e também os métodos contidos nela. Isso é assim porque na subclasse "Carnivoro" não tem um "constructor" próprio declarado e nem métodos.

Já quando a subclasse declara seu próprio "constructor", primeiro ela declara os parametros herdados no seu "constructor". Mas para que ela receba e use as propriedades de herança da classe pai, devemos declarar o método reservado do javaScript "super()". Dentro do método "super()" declaramos os parâmetros herdados da classe pai. Assim os parâmetros herdados serão válidos e só então a depois disso a subclasse poderá declarar atributos para propriedades próprias.

        
            class Carnivoro extends Mamiferos {
                constructor(especie,nome,idade,comePessoas) {
                    super(especie,nome,idade);
                    this.comePessoas = comePessoas;
                }
            }

Herança Prototipada em javaScript

A herança prototípica é a base do sistema de objetos do JavaScript. Cada objeto em JavaScript tem uma referência interna a outro objeto, chamado de “protótipo”. Quando uma propriedade ou método é acessado em um objeto, o JavaScript primeiro verifica se essa propriedade ou método existe no próprio objeto. Se não for encontrado, o JavaScript procurará no protótipo do objeto. Esse processo continua até que a propriedade ou método seja encontrado ou até que a cadeia de protótipos termine.

A herança prototípica permite a reutilização de código e a hierarquia de objetos sem a necessidade de classes. Isso proporciona uma flexibilidade significativa, já que um objeto pode herdar diretamente de outro objeto e até mesmo modificar seu protótipo em tempo de execução.

No javaScript a sintaxe para que uma função constructor herde propriedades e métodos de outros objetos é da seguinte maneira:

        
            function MonitorDeClasse(nome,email){
                Estudante.call(this, nome, email)
            }

O método "call()" está invocando o construtor da função "Estudante()" para dentro do escopo da função "MonitorDeClasse()" onde é armazenado o objeto que a função "MonitorDeClasse()" irá criar. Para isso acontecer declaramos o "this" (que referencia o "constructor" da função "Estudante()") como argumento para o método "call()" invocado. Já no segundo argumento em diante do método "call()" invocado, pode receber os parâmetros da função "Estudante()" que serão atribuidos para a função "MonitorDeClasse()"

Também um Objeto pode receber um "protótipo novo" criado apartir de outro protótipo de outro objeto como herança protótipal. Em javaScript usamos a seguinte sintaxe para isso:

        
            MonitorDeClasse.prototype = Object.create(Estudante.prototype);

Encapsulamento em Programação Orientada a Objetos

O encapsulamento é um dos pilares da orientação a objetos. Por meio dele, é possível simplificar bastante a programação, bem como proteger informações sigilosas ou sensíveis. Dominar esse conceito é fundamental, uma vez que com ele é possível deixar o código mais legível, funcional e reutilizável. A vantagem de utilizarmos esta estrutura é que conseguimos tratar toda saída e entrada de dados em uma classe

Encapsulamento é o mecanismo utilizado dentro da Orientação a Objetos para restringir o acesso externo ao comportamento interno de um objeto . As entradas, os processamentos e as saídas de um objeto não influenciam os dos outros, pois os seus relacionamentos são apenas referenciados. O encapsulamento também serve para ajudar as variáveis a não conflitarem durante a execução de uma aplicação e também não poluir o escopo global do browser.

Encapsular os dados de uma aplicação significa evitar que estes sofram acessos indevidos. Para isso, é criada uma estrutura que contém métodos que podem ser utilizados por qualquer outra classe, sem causar inconsistências no desenvolvimento de um código.

Na prática, isso é feito por meio de dois métodos: os getters e os setters.

  • Getters: a funcionalidade de um getter é obter um valor de uma variável declarada como privada e permitir a leitura apartir de fora da classe. Os métodos gettes podem ser programados para exibir os dados em formato apropriados. um get é declarado dentro da própria classe.
  • Setters: Um setter é um método que permite modificar o valor de uma atributo da classe, geralmente um atributo que esteja privado e que não seja acessível diretamente. Assim teremos mais controle sobre como os valores são atribuidos aos campos. Os métodos setters podem ser programados para validar seus argumentos e rejeitar quaisquer tentativa de inserção de dados inválidos,

Na linguagem javaScript devemos tomar um cuidado quando formos declarar estes métodos nas classes. Algo que não pode aconceter é nomear "getter" e "setter" com o mesmo nome de atributo da classe pois gera um erro.

Em Programação Orientada a Objeto o encapsulamento dispõe de niveis de proteção dos dados de uma objeto, que são três: "nivél publico", "nivél protegido", "nivél privado".

  • Nivél publico: consiste no nível de proteção mais baixo de todos, uma vez que todas as classes de um programa podem acessar os dados e os métodos livremente. Pode-se dizer que, nesse caso, não existe encapsulamento;
  • Nivél protegido: nesse nível, o acesso passa a ser mais restrito. Somente as classes derivadas da original é que têm total acesso aos atributos e métodos. Aqui, já existe encapsulamento, de modo que os métodos get e set passam a ser usados, bem como na declaração do atributo usa-se o private em vez do public;
  • Nivél privado: aqui, o acesso aos dados é feito somente dentro da própria classe onde ela foi declarada.

Já em javaScript só existeem dois niveis de proteção dados de uma objeto: nivél público e nivel privado.

Difetente de outras linguagens de programação que ao declarar atributos de propriedades em uma classe é necessário especificar o nivél de proteção coma palavra reservada "public", no javaScript os atributos das propriedades já estão implícitamente declarados como dados de nivel público, não sendo necessário específicar.

Para tornar uma propriedade de uma classe em nivél privado de proteção em javaScript atualmente, podemos declarar os nomes de atributos precedidos de "#" para torna-los privados. Assim só será possível acessar esses atribudos através dos métodos "get"" e "set".

        
            class pessoa {
                #nome;//nomes dos atributos estão fora do escopo do constructor
                #idade;
                constructor(nome, idade) {  //constructor da classe
                    this.#nome = nome;
                    this.#idade = idade;
                }
                get nome(){
                    return this.#nome;
                }
                set idade(idade) {
                    return this.#idade = idade;
                }
            }

No exemplo de sintaxe acima os métodos get e set foram declarados para acessar propriedades privadas da classe. O get tem funcionalidade de retornar a propriedade que esta privada. O set permite acessar a propriedade para podermos atribuir novo valor para ela através dos parâmetros do método criado. No JavaScript, para que os métodos get e set tenho acesso aos atributos modificados para privado de uma classe de fato, devemos declarar o nome do atributo fora do escopo do "constructor" da classe, caso contrário os métodos não terão acesso as propriedades da classe.

Polimorfismo em Programação Orientada a Objetos

Em programação orientada a objetos existe o conceito do polimorfismo (a palavra polimorfismo significa mùltiplas formas). O polimorfismo remete ao método existente em um objeto pai, que quando herdado para um outro objeto filho, pode mudar sua funcionalidade mesmo tendo o nome e assinatura iguais aos que foram declarados no objeto pai. Em outras palavras, permite que o mesmo nome de um método represente vários comportamentos diferentes.

Um ponto importante sobre métodos que devemos entender é: o que é uma assinatura de um método? A assinatura de um método é formada pelo seu nome e pelo tipo, quantidade e ordem de seus parâmetros. Desta forma, é possível que existam na mesma classe, métodos com o mesmo nome, desde que tenham listas de parâmetros diferentes.

        
            calcMedia(number,number){
                return //alguma expressão
            }
            calcMedia(number,number,number){
                return //outra expressão
            }

No exemplo de sintaxe acima declaramos dois métodos onde eles compartilham o mesmo none, o mesmo tipo de parâmetro (number), mas a quantidade de parâmetros é diferente entre eles pois o primeiro método possui dois parâmetros e o segundo método possui três parâmetros. portanto as assinaturas entres eles são diferentes.

Tipos de polimorfismo

Na proramação orientada a objetos existem vários tipos de polimorfismo, porém os dois tipos mais importantes e mais usados são: polimorfismo sobrescrito e polimorfismo de sobrecarga.

Polimorfismo sobrescrito
Um método é do tipo sobrescrito quando ele é declarado mais de uma vez com mesmo nome, mesma assinatura. Porém a expressão de retorno é modificada no método que foi herdado, o que faz mudar a funcionalidade do método.
        
            class MamiferosTerrestres{
                constructor(especie){
                    this.especie = especie;
                }

                locomover(){
                    return console.log(`${this.especie} se locomove correndo`)
                }
            }

            class Cavalo extends MamiferosTerrestres{
                constructor(especie){
                    super(especie)
                }
                
                locomover(){
                    return console.log(`${this.especie} se locomove galopando`)
                }
            }
Polimorfismo de sobrecarga
A sobrecarga de métodos ocorre quando temos vários métodos com o mesmo nome mas com parâmetros diferentes.
        
            class Calculadora {
                constructor(valor1, valor2){
                   this.valor1 = valor1;
                   this.valor2 = valor2;
                }
             
                calcMedia(number){
                return ((this.valor1*number) + (this.valor2*number)) / 2;
                }
                
                calcMedia(number=2){
                   return ((this.valor1*number) + (this.valor2*number)) / 2;
                   }
             }
