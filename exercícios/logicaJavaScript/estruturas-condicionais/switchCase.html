Estrutura de decisão switch...case

A instrução switch avalia uma expressão , comparando o valor da expressão com uma série de case cláusulas e executa instruções após a primeira case cláusula com um valor correspondente, até que uma instrução break seja encontrada. A cláusula default de uma instrução switch será saltada se não casecorresponder ao valor da expressão.

Sintaxe

        
            switch  ( expressão )  { 
                caso  valor1 :
                  declarações
                caso  valor2 :
                  declarações
                // …
                 valor do caso N  :
                  declarações
                padrão :
                  declarações
              }

Uma instrução switch primeiro avalia sua expressão. Em seguida, ela procura a primeira case cláusula cuja expressão avalia o mesmo valor que o resultado da expressão de entrada (usando a comparação de igualdade estrita ) e transfere o controle para essa cláusula, executando todas as instruções após essa cláusula. Os valores de cláusula são avaliados apenas quando necessário — se uma correspondência já for encontrada, caseos valores de cláusula subsequentes não serão avaliados, mesmo quando forem visitados por fall-through .

Se nenhuma case cláusula correspondente for encontrada, o programa procura a cláusula opcional default e, se encontrada, transfere o controle para essa cláusula, executando instruções após essa cláusula. Se nenhuma cláusula default for encontrada, o programa continua a execução na instrução após o final de switch. Por convenção, a cláusula default é a última cláusula, mas não precisa ser assim. Uma declaração switch pode ter apenas uma cláusula default; várias cláusulas default resultarão em um arquivo

Instrunção break

Podemos usar a instrução break dentro de uma instrução do corpo switch para interromper a execução antecipadamente, geralmente quando todas as instruções entre duas cláusulas case foram executadas. Se break for omitido, a execução continuará na primeira instrução switch seguinte. A execução prosseguirá para a próxima cláusula case, mesmo para a cláusula default, independentemente de o valor dessa cláusula corresponder. Esse comportamento é chamado de "fall-through".

Escopo léxico

As cláusulas case e default são como rótulos : elas indicam possíveis lugares para os quais o fluxo de controle pode ir. No entanto, eles próprios não criam escopos lexicais (nem se rompem automaticamente — como demonstrado acima). Por exemplo:

        
            const ação = "dizer_olá"; 
            switch( ação )  { 
              case "dizer_olá":
                const mensagem = "olá"; 
                console.log( mensagem ); 
                break ; 
              case "dizer_oi" :
                const mensagem = "oi"; 
                console.log( mensagem ); 
                break ; 
              padrão : 
                console.log ("Ação vazia recebida." ); 
            }

Este exemplo produzirá o erro "Uncaught SyntaxError: Identifier 'message' has been declareed", porque a primeira declaração const message = 'olá', conflita com a segunda declaração const message = 'oi', mesmo quando eles estão dentro de suas próprias cláusulas case separadas. Por fim, isso se deve ao fato de ambas constas declarações estarem dentro do mesmo escopo de bloco criado pelo switchcorpo. Para corrigir isso, sempre que precisar usar letou constdeclarações em uma casecláusula, envolva-os em um bloco {}.

Agora este código esta exibindo "olá" no console.log() como deveria, sem nenhum erro.
